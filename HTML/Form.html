<form>태그 속성(폼안에 폼은 안된다.)

method 속성 : 사용자가 입력한 내용들을 서버 쪽 프로그램으로 어떻게 넘겨줄지 지정
get : 주소 표시줄에 사용자가 입력한 내용이 그대로 드러남. 256~2,096byte까지 전송 가능
post : 사용자의 입력을 표준 입력(standard input)방식으로 넘겨줌. 입력길이에 제한이 없고 주소창에 입력값이 드러나지 않음
action 속성 : <form>태그의 내용을 서버로 넘겼을 때 내용을 처리해 줄 서버상의 프로그램을 지정

<label>태그와 속성

  • 폼 요소에 레이블(텍스트)을 붙이는 태그
  • 라디오 버튼이나 체크 박스에서 텍스트 부분을 클리해도 선택가능.
  • for 속성 : label의 연결을 위해 특정 값을 지정.

<fieldset>태그

  • 폼 요소를 그룹으로 묶는 태그

<legend>태그

  • 그룹으로 묶는 구역에 제목을 붙이는 태그

<input>태그와 속성

  • 사용자가 입력하는 부분
  • title 속성 : 해당 부분에 마우스 커서를 올리고 대기하면, 추가적인 정보 표시.
  • type 속성 : 입력하는 내용의 종류
    • hidden : 화면 상의 폼에는 보이지 않는다.
      폼을 서버로 전송할 때 서버로 함께 전송된다.
    • text : 한 줄짜리 텍스트 입력 필드.
      주로 아이디나 이름, 주소 등 텍스트 입력
    • password : 비밀번호 입력 필드.
      입력내용이 '*'나 '·'로 표시
    • text, password 필드 속성
      • name : 텍스트 필드를 구별할 수 있도록 이름을 붙힘
      • size : 텍스트 필드의 길이 지정. 화면에 몇 글자가 보일지 지정
      • value : text 필드에서만 존재. 텍스트 필드 부분안에 표시될 내용. 없으면 빈칸.
      • maxlength : 텍스트 필드에 입력할 수 있는 최대 문자 개수 지정.
      • mixlength : 입력해야 할 최소 글자.
    • search : 검색 필드. 검색 창에 x 표시가 되어 검색어 삭제 쉬움
    • email : 메일 주소 입력 필드. 메일 주소 형식 자동 체크
    • url : 웹 주소 필드. "http://" 로 시작하는 사이트 주소 입력
    • tel : 전화번호 입력 필드. 사용자 입력을 체크하지는 않음.
    • number : 숫자 입력 필드. 브라우저에 따라 스핀 박스로 표시
    • range : 숫자 입력 필드. 슬라이드 막대로 숫자 입력.
    • number, range 필드 속성
      • min : 최소값
      • max : 최대값
      • value : 기본 값
      • step : 바뀌는 수의 크기
    • radio : 여러 항목 중 하나만 선택할 때
    • checkbox : 여러 항목 중 둘 이상을 선택할 때
    • radio, checkbox 필드 속성
      • name : 라디오 버튼이나, 체크박스를 구분하기 위해 이름 지정.
        라디오 버튼은 관련 있는 그룹끼리는 name 속성 값을 똑같이 만듬.
      • value : 선택한 라이도 버튼이나 체크박스를 서버로 알려 줄 때 넘길 값 지정. 필수 속성.
      • checked : 라디오 버튼의 항목들 중 기본으로 선택해 놓을 항목이 있다는 사용.
    • color : 색상 표에서 색상 선택
    • date : 날짜 선택 "yyyy-mm-dd"형식.
    • month : 월과 연도 선택 "yyyy-mm"
    • week : 주와 연도 선택
    • time, datetime-local : 시간입력
    • time, datetime-local 필드 속성
      • min : 날짜나 시간의 최솟값 ex time)min="20:00" ex datetime-local)min="2000-10-06T22:22:55"
      • max : 날짜나 시간의 최댓값 ex time)max="22:00" ex datetime-local)max="2014-11-16T21:25:33"
      • step : 날짜나 시간 조절 단위 ex)step="300"(5분)
      • value : 화면에 표시할 초기값 지정. time일 경우, 시간은 00:00~25:59로 입력.
        datetime 이나 datetime-local일 경우, 날짜 다음에 T를 쓰고 24시간제로 시간을 지정.
    • submit : 폼 전송 버튼. 사용자 입력 내용을 서버로 전송. value 속성을 이용해 버튼 표시 내용 지정
    • reset : 사용자 입력 내용 전부 삭제. value 속성을 이용해 버튼 표시 내용 지정
    • image : submit 버튼 대신 이미지 삽입.
  • id 속성 : 여러 번 사용된 폼 요소를 구분하기 위해 사용
    • <label>태그를 이용해 캡션을 붙일 수 있다.
    • CSS를 이용해 각 요소마다 다른 형태로 꾸밀 수 있다.
  • autofocus 속성 : 페이지를 불러오자마자 원하는 폼 요소에 마우스 커서 표시
  • placeholder 속성 : 입력란에 표시하는 힌트. 필드 클릭시 사라짐.
  • readonly 속성 : 내용을 보기만 하고 입력하지 못하게 함.
  • reqired 속성 : 필수 필드 체크. 입력하지 않으면 오류.
  • list 속성 : <datalist>태그의 id 에 적용한 값을 이용하여 사용할 데이터 목록 지정

<datalist>태그

  • 데이터 목록에 제시한 값 중에서 선택하면 그 값이 자동으로 텍스트 필드에 입력됨.
  • 데이터 목록에 id를 이용해 이름을 붙이고, <input>태그의 list 속성에 데이터 목록 id를 지정

<select>, <optgroup>태그

  • 여러 옵션 중에서 선택 - 드롭다운 메뉴
  • 공간을 최소한으로 사용하면서 여러 옵션 표시 가능
  • <select> 태그 속성
    • size 속성 : 화면에 표시될 드롭다운 메뉴의 항목 개수 지정
    • multiple 속성 : 브라우저 화면에 여러 개의 옵션이 함께 표시.
      Ctrl 키를 누른 상태로 드롭다운 메뉴에 있는 여러 항목 선택 가능.
  • <option> 태그 속성
    • value 속성 : 옵션을 선택했을 때 서버로 넘겨질 값 지정.
    • selected 속성 : 화면에 표시될 때 기본으로 선택되어 있는 옵션 지정.
    • label 속성 : 사용자를 위해 브라우저에 표시할 레이블. 따로 지정하지 않을 경우 value값을 사용.

<option>태그

  • option의 여러 항목을 그룹으로 묶을 때 사용
  • label 속성을 사용해 그룹 제목을 붙임.

<textarea>태그

  • 텍스트 영역 - 여러 줄의 텍스트 입력
  • <textarea>태그 속성
    • name 속성 : 다른 폼 요소와 구분하기 위해 텍스트 영역의 이름 지정.
    • clos 속성 : 텍스트 영역의 가로 너비를 문자 단위로 지정.
    • rows 속성 : 텍스트 영역의 세로 길이를 줄 단위로 지정.

<button>태그

  • 다양한 형태의 버튼 삽입.
  • 화면 낭독기에서 버튼임을 정확히 전달할 수 있음.
  • CSS를 이용해 원하는 형태로 꾸밀 수 있음
  • <button>태그 속성
    • submit 속성 : 폼을 서버로 전송.
    • reset 속성 : 폼에 입력한 모든 내용 초기화.
    • button 속성 : 버튼 형태만 만들 뿐 자체 기능은 없음.

<output>태그

  • 계산 결과를 브라우저에 표시.
  • 브라우저 화면에 다르게 표시되는 것은 아니나,
    일반 텍스트가 아니라 계산의 결과값이라는 것을 웹 브라우저가 정확히 인식할 수 있음.

<progress>태그

  • 작업 진행 상태를 브라우저에 표시.
  • <progress>태그 속성
    • value 속성 : 작업 진행 상태를 나타내며, 부동 소수점으로 표현.
      이 값은 0보다 크거나 같고 max 값보다 적거나 같아야 함.
      만약 max 값이 지정되지 않았다면 이 값은 1.0보다 작아야 함.
    • max 속성 : 작업이 완료되려면 얼마나 많은 작업을 해야 하는지 부동소수점으로 표현.
      이 값은 0보다 커야 함.

<meter>태그

  • 전체 크기 중에서 얼마나 차지하는지를 표현할 때 사용.
  • <meter>태그 속성
    • min, max 속성 : 범위의 최솟값과 최댓값을 나타냄. 정하지 않으면 0과 1로 간주.
    • value 속성 : 범위 내에서 차지하는 값.
    • low 속성 : "이 정도면 낮다." 라고 할 정도의 값을 지정.
    • high 속성 : "이 정도면 높다." 라고 할 정도의 값을 지정.
    • optimum 속성 : "이 정도면 적당하다." 라고 할 정도의 범위를 지정.
      high보다 크면 value값이 클수록 좋고, low보다 작으면 value값이 작을수록 좋다.
